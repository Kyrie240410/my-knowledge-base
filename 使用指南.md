# 使用指南

## 日常写作工作流

### 方式一：直接在 GitHub 上编辑（最简单）

适合快速修改单篇文章。

1. 打开 GitHub 仓库
2. 进入 `docs/` 目录找到要编辑的文件
3. 点击右上角的铅笔图标（Edit）
4. 修改内容（支持 Markdown 预览）
5. 滚动到底部，填写提交信息
6. 点击 `Commit changes`
7. 等待 2-3 分钟自动部署

### 方式二：本地编辑后推送（推荐）

适合批量编辑或复杂修改。

```bash
# 1. 克隆仓库（首次）
git clone https://github.com/你的用户名/my-knowledge-base.git
cd my-knowledge-base

# 2. 安装依赖（首次）
npm install

# 3. 启动开发服务器
npm run docs:dev

# 4. 在浏览器中预览
# http://localhost:5173/my-knowledge-base/

# 5. 编辑 Markdown 文件
# 使用 VS Code 等编辑器

# 6. 构建检查
npm run docs:build

# 7. 提交更改
git add .
git commit -m "添加新笔记"
git push origin main

# 8. 自动部署（GitHub Actions）
```

### 方式三：语雀写作后导出

适合习惯在语雀写作的用户。

```bash
# 1. 获取语雀 Token
# 登录语雀 → 个人设置 → 开发者管理 → 生成 Token

# 2. 安装导出工具
npm install -g yuque-exporter

# 3. 导出文档
yuque-exporter --token=你的Token

# 4. 导出的文件在 ./storage/ 目录
# 复制到 docs/ 对应目录

# 5. 提交到 GitHub
git add .
git commit -m "从语雀导入笔记"
git push origin main
```

## 添加新页面

### 1. 创建 Markdown 文件

在对应目录创建 `.md` 文件：

```bash
# 示例：添加一篇前端笔记
touch docs/study/frontend/vue3-composition-api.md
```

### 2. 编写内容

```markdown
# Vue 3 Composition API 学习笔记

## 为什么需要 Composition API

在 Vue 2 中，我们使用 Options API：

```javascript
export default {
  data() {
    return {
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}
```

## 基本用法

```vue
<script setup>
import { ref, computed } from 'vue'

const count = ref(0)
const double = computed(() => count.value * 2)

function increment() {
  count.value++
}
</script>
```

## 总结

Composition API 的优势：
1. 更好的逻辑复用
2. 更灵活的代码组织
3. 更好的 TypeScript 支持
```

### 3. 添加到导航

编辑 `docs/.vitepress/config.mjs`：

```javascript
sidebar: {
  '/study/frontend/': [
    {
      text: '前端技术',
      items: [
        { text: 'Vue 3 组合式 API', link: '/study/frontend/vue3-composition-api' },
        // ... 其他链接
      ]
    }
  ]
}
```

### 4. 提交部署

```bash
git add .
git commit -m "添加 Vue 3 Composition API 笔记"
git push origin main
```

## 搜索功能

VitePress 内置本地搜索，无需配置。

使用方式：
- 按 `Ctrl + K`（Mac: `Cmd + K`）打开搜索
- 或点击导航栏的搜索图标

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + K` | 打开搜索 |
| `Ctrl + Shift + K` | 打开搜索（新窗口）|
| `Ctrl + /` | 聚焦搜索框 |

## 最佳实践

1. **定期整理**：每周回顾一次，归档过期内容
2. **保持链接有效**：删除页面时检查是否有其他页面链接到它
3. **使用相对路径**：`./file.md` 而非绝对路径
4. **添加 frontmatter**：为重要页面添加描述信息
   ```markdown
   ---
   title: 页面标题
   description: 页面描述
   ---
   ```
5. **及时提交**：小步快跑，频繁提交

## 备份策略

由于使用 Git，天然具有版本控制：

```bash
# 查看历史
git log --oneline

# 回滚到某个版本
git reset --hard 提交哈希

# 创建分支备份
git branch backup-2024-01-01
```

同时建议：
- 开启 GitHub 的仓库备份功能
- 定期导出为 PDF（可选）
