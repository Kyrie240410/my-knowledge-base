# 部署指南

## 快速部署到 GitHub Pages

### 第一步：创建 GitHub 仓库

1. 登录 [GitHub](https://github.com)
2. 点击右上角 `+` → `New repository`
3. 仓库名称填写：`my-knowledge-base`
4. 选择 `Public`（公开）
5. 点击 `Create repository`

### 第二步：推送代码到 GitHub

在本地项目目录执行：

```bash
# 添加远程仓库（替换为你的用户名）
git remote add origin https://github.com/你的用户名/my-knowledge-base.git

# 推送代码
git push -u origin main
```

### 第三步：配置 GitHub Pages

1. 打开 GitHub 仓库页面
2. 点击 `Settings`（设置）
3. 左侧菜单选择 `Pages`
4. `Source` 选择 `Deploy from a branch`
5. `Branch` 选择 `gh-pages` / `root`
6. 点击 `Save`

### 第四步：修改配置（重要）

编辑 `docs/.vitepress/config.mjs`：

```javascript
export default defineConfig({
  // 修改为你的 GitHub 用户名
  base: '/my-knowledge-base/',
  
  themeConfig: {
    // 修改为你的 GitHub 信息
    socialLinks: [
      { icon: 'github', link: 'https://github.com/你的用户名' }
    ],
    
    editLink: {
      pattern: 'https://github.com/你的用户名/my-knowledge-base/edit/main/docs/:path'
    }
  }
})
```

编辑 `docs/index.md`：

```markdown
actions:
  - theme: brand
    text: GitHub
    link: https://github.com/你的用户名/my-knowledge-base
```

### 第五步：触发部署

```bash
# 提交修改
git add .
git commit -m "config: 更新部署配置"
git push origin main
```

GitHub Actions 会自动构建并部署。

### 第六步：访问你的知识库

等待 2-3 分钟后访问：

```
https://你的用户名.github.io/my-knowledge-base
```

## 自定义域名（可选）

### 购买域名

推荐平台：
- [阿里云](https://wanwang.aliyun.com/)
- [腾讯云](https://dnspod.cloud.tencent.com/)
- [Namecheap](https://www.namecheap.com/)（国外）

### 配置 DNS

1. 在域名管理后台添加 CNAME 记录：
   - 主机记录：`www` 或 `@`
   - 记录类型：`CNAME`
   - 记录值：`你的用户名.github.io`

2. 在仓库创建 `CNAME` 文件：
   ```bash
   echo "your-domain.com" > docs/CNAME
   git add docs/CNAME
   git commit -m "添加自定义域名"
   git push origin main
   ```

3. 在 GitHub Pages 设置中配置自定义域名

## 自动化部署

项目已配置 GitHub Actions，每次推送到 `main` 分支会自动：

1. 安装依赖
2. 构建网站
3. 部署到 `gh-pages` 分支
4. 更新 GitHub Pages

### 查看部署状态

1. 打开 GitHub 仓库
2. 点击 `Actions` 标签
3. 查看工作流运行状态

## 常见问题

### Q: 部署后页面空白？

**A**: 检查 `base` 配置是否正确：
```javascript
// 如果仓库名是 my-knowledge-base
base: '/my-knowledge-base/',

// 如果仓库名是 username.github.io
base: '/',
```

### Q: 样式丢失？

**A**: 确保 `base` 配置与仓库名一致，且路径以 `/` 开头和结尾。

### Q: 如何更新内容？

**A**: 
1. 编辑 Markdown 文件
2. 提交到 GitHub
3. 自动部署（约 2-3 分钟）

```bash
git add .
git commit -m "更新笔记"
git push origin main
```

### Q: 如何添加新页面？

**A**:
1. 在对应目录创建 `.md` 文件
2. 在 `config.mjs` 的 sidebar 中添加链接
3. 提交并推送
